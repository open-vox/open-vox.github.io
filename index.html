Abstract

The reconstruction and understanding of the surrounding environment is crucial for mobile robots to execute language-based human instructions. In recent years, open-vocabulary mapping enhanced by visual language models (VLMs) has become a common solution. While integrated object understanding helps mitigate semantic ambiguity in point-wise feature maps, efficiently obtaining inferential semantic understanding and robust incremental reconstruction at the instance-level remains challenging. To address these challenges, we introduce OpenVox, a real-time incremental open-vocabulary instance-level probabilistic voxel representation. In the front-end, we design a fast instance segmentation and comprehension pipeline that enhance language reasoning through encoding caption. In the back-end, we design instance-level probabilistic voxels and model the cross-frame incremental fusion problem as two subtasks: Maximum Likelihood Estimation (MLE) for instance association and Maximum A Posteriori estimation (MAP) for map updating, ensuring robustness to sensor and segmentation noise. Results across multiple datasets show that OpenVox achieves superior performance in zero-shot instance segmentation, semantic segmentation, and open-vocabulary instance retrieval. Additionally, real-world robotics experiments confirm OpenVox's strong potential for stable, real-time operation.

Framework

In the frontend, the Instance Segmentation & Understanding module implements an efficient pipeline for instance-level semantic understanding, powered by caption encoding. It processes RGB image frames to generate 2D instance segmentation masks and their corresponding semantic annotations. In the backend, we project the 2D masks onto a 3D map using probabilistic methods and perform probabilistic updates. This process is modeled as two subtasks: instance association and map update. The first subtask involves associating instances from the observed masks and maps by solving the MLE problem, while the second subtask updates the instance voxel maps by solving the MAP problem.
